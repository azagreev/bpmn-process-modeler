# Manufacturing Patterns — производственные процессы

Читается на Step 2, когда domain определён как manufacturing, production, MES integration.

---

## 1. Production order

1. Создание заказа на производство — User Task (ПДО — Планово-диспетчерский отдел)
2. Проверка наличия материалов — Business Rule Task (сверка с WMS / ERP)
3. Запуск в производство — Service Task (интеграция с MES — Manufacturing Execution System)
4. Цепочка операций — обычно подпроцессы по рабочим центрам: механическая обработка / термообработка / сборка / покраска
5. Контроль качества — User Task
6. Упаковка и передача на склад готовой продукции — Send Task

**Особенности моделирования:**
- Часто нужны inclusive gateways (OR) — параллельные операции, которые могут выполняться в разном порядке и не все обязательны (например, покраска не для всех артикулов)
- Для non-stop производства (металлургия, химия) — batch-логика вместо discrete: другие паттерны
- Интеграция с MES — через external worker pattern в Camunda 7 (task topics: mes.start, mes.complete, mes.reject)

**Compliance:** ISO 9001 (QMS), для пищевой / фармы — дополнительно ISO 22000 / GMP; для автопрома — IATF 16949; для оборонки — ГОСТ РВ 0015-002.
