# HR Patterns — процессы управления персоналом

Читается на Step 2, когда domain определён как HR, employee onboarding, leave, compensation, performance review.

---

## 1. Онбординг

1. Offer accepted — Message Start Event (кандидат подписал offer letter)
2. Параллельный шлюз:
   - Подготовка рабочего места — Service Task (IT: ноутбук, доступы, учётные записи)
   - Оформление документов — User Task (кадры: трудовой договор, заявления, СНИЛС, ИНН, банковские реквизиты)
   - Назначение ментора — User Task (руководитель)
   - Заведение в 1С ЗУП / внутренние системы — Service Task
3. Первый день — User Task (приветственная встреча, welcome pack)
4. Испытательный срок — sub-process с timer (3 месяца) и промежуточными чек-поинтами (30 / 60 / 90 дней)

**Compliance:** ТК РФ ст. 57 (содержание трудового договора), ст. 70 (испытательный срок — максимум 3 месяца для обычных сотрудников, 6 для руководителей).

**Особенности моделирования:**
- Параллельный шлюз на шаге 2 — classic AND split, все задачи обязательны до старта
- Шаг 4 часто отдельная диаграмма (event subprocess): по каждому чек-поинту — решение продолжить / расстаться

---

## 2. Отпуск

Классический approval loop. Сотрудник подаёт заявление → руководитель согласует → HR оформляет приказ → выплата отпускных.

Референс для BPMN-паттерна: `bpmn-patterns.md` #1 (Approval loop).

**Compliance:** ТК РФ ст. 114–127 (оплачиваемые отпуска); ст. 136 (выплата отпускных не позднее чем за 3 дня до начала отпуска).
