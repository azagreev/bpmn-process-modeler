# Public Sector Patterns — государственные услуги

Читается на Step 2, когда domain определён как public sector, government services, МФЦ, Госуслуги, межведомственное взаимодействие.

---

## 1. Государственная услуга

1. Подача заявления — User Task (гражданин через портал Госуслуг / МФЦ / лично в ведомство)
2. Проверка полноты документов — User Task (специалист)
3. Межведомственные запросы — параллельный шлюз + несколько Send + Receive Task (СМЭВ — система межведомственного электронного взаимодействия)
4. Принятие решения — User Task (должностное лицо)
5. Выдача результата — Send Task (электронный документ в ЛК) / User Task (бумажный документ в МФЦ)

**Compliance-аннотации:**
- 210-ФЗ «Об организации предоставления государственных и муниципальных услуг» — базовая рамка
- Регламент конкретной услуги — устанавливается ведомством
- Сроки: типично 10–30 рабочих дней в зависимости от услуги (аннотация на диаграмме с точной ссылкой на регламент)
- Межведомственный запрос по СМЭВ — не более 5 рабочих дней на ответ

**Особенности моделирования:**
- Часто нужен collaboration pool: «Гражданин» + «МФЦ / ведомство» + «Другие ведомства (через СМЭВ)»
- Timer Boundary на каждом шаге с регламентным сроком
- Если гражданин не добрал документов — non-interrupting timer + notification с просьбой донести
