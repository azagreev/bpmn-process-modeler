# Healthcare Patterns — медицинские процессы

Читается на Step 2, когда domain определён как healthcare, medical, pharma.

Универсальная рамка комплаенса: HIPAA (США), GDPR + EHDS (ЕС), 152-ФЗ + приказы Минздрава (РФ). Все задачи с доступом к медицинским данным — с аннотацией о принципе минимально необходимого доступа (least privilege / minimum necessary).

---

## 1. Приём пациента (амбулаторно)

1. Регистрация — User Task (регистратура)
2. Анамнез — User Task (мед. сестра)
3. Осмотр врача — User Task (врач-специалист)
4. Назначения — параллельный шлюз (анализы + процедуры + рецепты)
5. Выписка / завершение приёма — End Event

**Compliance-аннотации:**
- Согласие пациента на обработку ПДн (152-ФЗ в РФ, HIPAA Notice of Privacy Practices в США, GDPR Art. 9 для sensitive data в ЕС)
- Ведение мед. карты (электронной / бумажной) по приказу Минздрава
- Для передачи данных третьим лицам (страховая, другие врачи) — отдельное согласие

**Особенности моделирования:**
- Пул обычно «Клиника», с лэйнами «Регистратура» / «Мед. сестра» / «Врач» / «Лаборатория»
- Шаг 4 часто expanded в sub-process с несколькими параллельными ветками

---

## 2. Диспансеризация

Timer Start Event (напоминание раз в год по возрастной группе) + последовательность обследований + агрегирующий Business Rule Task для финального заключения и группы здоровья.

**Compliance:** Приказ Минздрава РФ о порядке диспансеризации (группа здоровья I–IIIб, сроки, содержание).
